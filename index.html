프로모션 시뮬레이터

기획 중인 행사 물량을 얹었을 때, 현재고로 행사 기간을 버틸 수 있는지 배수 시나리오로 즉석 점검합니다.

시뮬레이션 조건

배수 = 행사 기간 중 평소 일평균 대비 몇 배 팔릴지의 가정입니다. 정확한 물량을 몰라도 "이 정도 행사면 2배쯤"으로 판단할 수 있게, 세 가지 시나리오를 동시에 보여줍니다. 행사 중 하루 소비 = 평소 일평균 × 배수. 초록=완주 후에도 안전재고 이상 여유, 노랑=완주하지만 빠듯(안전재고 밑), 빨강=행사 기간 중 소진(추가 발주 필요).
품목을 추가하면 시나리오별 결과가 표시됩니다.

수요 예측 시뮬레이터

영업 전망(예측 배수)과 SCM 조율안(조율 배수)을 넣어, 수요가 바뀌면 발주 기준이 어떻게 변하는지 비교합니다. 조율안은 실제 발주에 반영할 수 있습니다.

품목 추가

품목을 추가하면 현재(실측) / 예측(영업) / 조율(SCM) 세 기준의 일평균·안전재고·ROP·목표재고·발주량을 나란히 보여줍니다. 조율 배수 기본값은 예측의 80%로 제안하며 직접 조정할 수 있습니다. 낙관적 영업 전망을 SCM이 보수적으로 조율한 값이 실제 발주 기준이 됩니다.
품목을 추가하면 현재·예측·조율 비교가 표시됩니다.

설정

계산 파라미터의 기본값입니다. 저장하면 다음부터 자동 적용됩니다.

계산 파라미터

계산은 일별 방식입니다 — 재고수불부의 일자별 배송을 최근 6개월(183일) 사용하며, 평소(중앙값)의 10배를 넘는 대량 출고일(해외보충·생산 등)과 빈 일자행은 σ 계산에서 자동 제외합니다. 안전재고 = Z×σ(일별)×√리드타임, ROP = 일평균×리드타임 + 안전재고, 목표재고 = 일평균×(리드타임+발주주기) + 안전재고.
발주주기: 한 번 발주 시 며칠치를 채울지(예: 60일이면 발주 후 약 60일간은 발주 0). 검토주기: 재고를 들여다보는 점검 간격(계산엔 미반영, 안내용). 위 집계 선택은 대시보드 추세·상승 판정의 표시용 구간입니다.

데이터 백업 · 이동

기기를 옮길 때(데스크톱 ↔ 노트북) 사용합니다. 한쪽에서 백업 내보내기로 파일을 저장하고, 다른 기기에서 불러오기로 복원하세요. 회의 이력·발주확정·설정이 모두 포함됩니다.
